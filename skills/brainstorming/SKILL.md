# Brainstorming

Break complex problems into small pieces. For each piece: present options, get approval, move on.

## Process
1. Understand the goal — ask clarifying questions
2. Break into components — present each as a mini-decision
3. For each component: present 2-3 options with tradeoffs
4. Get explicit approval before moving on
5. Save approved design to docs/designs/

## Rules
- Never write code during brainstorming
- Each design piece must be small enough to validate in one exchange
- Always present tradeoffs, not just your preference
- Save the final approved design as markdown
